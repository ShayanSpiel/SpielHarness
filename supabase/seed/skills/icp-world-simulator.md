# ICP World Simulator

Translates source material (session log or topic text) from build-log language into ICP language.

## The 5 steps
1. Build the Reader — one specific ICP, identity-rich
2. Build the Belief, Pain, Point triad
3. Build the Proof — 3 concrete facts
4. Build the Meaning — the aha, first-person
5. Pick the example_pattern

## Translation rules
- Use ICP vocabulary from the strategy files (voice, audience, methodology)
- Pain is a recognizable pattern, not a fabricated scene
- Point must contradict Belief
- Meaning is first-person, ICP voice — the sentence the reader remembers 24h later

## Output fields
reader, pain, belief, point, proof, meaning, example_pattern, axis
